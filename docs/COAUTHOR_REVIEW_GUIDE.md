# Coauthor Review Guide

Guia breve para coautores que quieren evaluar la app sin leer primero todo el codigo.

Actualizado: `2026-05-02`

## 1. Que mirar primero

- app publica principal: [https://dice.sonar2026.es](https://dice.sonar2026.es)
- app Railway: [https://app-production-4b8d.up.railway.app](https://app-production-4b8d.up.railway.app)

## 2. Codigos recomendados para revisar

- `CTRL1234`
  recorrido demo control
- `NORM0000`
  recorrido demo `norm_0`
- `NORM0001`
  recorrido demo `norm_1`

## 3. Diseno experimental que debes asumir

El unico diseno vigente es:

- `control`
- `norm_0..norm_60`

La norma social es fija por tratamiento y no se actualiza dinamicamente durante la sesion.

## 4. Feedback especialmente util

- claridad del copy
- coherencia entre pantallas
- si queda claro que solo cuenta la primera tirada
- si la norma social se entiende sin sonar manipuladora
- fricciones de comprension
- problemas responsive
- tono general del experimento

## 5. Flujo a revisar

1. idioma
2. consentimiento
3. instrucciones
4. comprension
5. dado
6. reporte
7. revelado final
8. ganador o no ganador
9. cobro o cierre final

## 6. Preguntas orientativas

- Se entiende que solo cuenta la primera tirada?
- La pregunta de comprension coincide con las instrucciones?
- El mensaje social parece claro pero no agresivo?
- La pantalla de pago es creible y sobria?
- Hay algun punto donde el usuario pueda pensar que el sistema cambia lo que ve a mitad de decision?
